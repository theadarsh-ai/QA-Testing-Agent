Build a full-stack AI agent application called DesignGuard — a Visual 
Accessibility QA Agent that watches any website screenshot, detects WCAG 
accessibility violations using Gemini multimodal vision, and autonomously 
fixes them using browser DevTools injection actions. This is a submission 
for the Gemini Live Agent Challenge UI Navigator track.

---
TECH STACK
- Backend: Python, FastAPI, LangGraph (Google GenAI SDK)
- Frontend: Streamlit (dark professional UI)
- AI: Google Gemini 2.0 Flash via google-generativeai SDK (multimodal)
- Database: Google Firestore (scan history, violation reports)
- Deployment: Google Cloud Run
- Agent Framework: LangGraph StateGraph

BACKEND — main.py
FastAPI app with these endpoints:

POST /scan
- Accepts: user_id (str), url (str), screenshot_base64 (str)
- Calls agent.run_scan_loop()
- Returns: scan_id, violations_found (list), fixes_applied (list),
  compliance_score_before (int), compliance_score_after (int)

POST /apply-fix
- Accepts: scan_id (str), violation_id (str)
- Returns: action_type, target_element, fix_code, explanation

GET /history/{user_id}
- Returns: last 10 scans from Firestore

GET /health
- Returns: {"status": "ok"}

Enable CORS for all origins.

BACKEND — gemini_vision.py

Function: analyze_screenshot(screenshot_base64: str) -> dict

Use google-generativeai SDK. Model: gemini-2.0-flash.
Convert base64 to PIL Image. Pass to Gemini multimodal with this system prompt:

"You are DesignGuard, an expert AI accessibility auditor. You analyze 
website screenshots and detect WCAG 2.1 accessibility violations using 
only visual inspection — no code access, no DOM.

Detect ALL violations across these categories:
1. COLOR_CONTRAST — Text/background contrast below 4.5:1 (AA)
2. FONT_SIZE — Text below 12px minimum
3. MISSING_ALT_TEXT — Images with no descriptive context
4. KEYBOARD_FOCUS — Interactive elements with no focus indicator
5. BUTTON_SIZE — Touch targets smaller than 44x44px
6. HEADING_STRUCTURE — Missing or illogical heading hierarchy
7. LINK_TEXT — Generic link text like 'click here'
8. FORM_LABELS — Input fields with no visible label

Respond in EXACT JSON:
{
  'violations': [
    {
      'violation_id': 'unique_id',
      'category': 'COLOR_CONTRAST',
      'severity': 'critical' | 'serious' | 'moderate',
      'element_description': 'visual location of element',
      'wcag_criterion': 'WCAG 1.4.3',
      'current_value': 'what it currently is',
      'required_value': 'what it should be',
      'fix_description': 'what needs to change',
      'fix_action': {
        'type': 'css_injection' | 'attribute_injection' | 'devtools_command',
        'property': 'CSS property or HTML attribute',
        'old_value': 'current value',
        'new_value': 'corrected value',
        'devtools_command': 'exact JavaScript for browser console'
      }
    }
  ],
  'compliance_score': 0-100,
  'page_summary': 'brief description',
  'total_violations': integer,
  'critical_count': integer,
  'serious_count': integer,
  'moderate_count': integer
}"

Add try/except for JSON parsing errors.

BACKEND — agent.py (LangGraph)

LangGraph StateGraph with these nodes:
1. observe — receives screenshot + url, prepares state
2. detect — calls gemini_vision.analyze_screenshot()
3. prioritize — sorts by severity (critical first), selects top 5
4. fix — calls fix_generator.generate_fix_action() for each
5. verify — recalculates compliance score after fixes
6. report — compiles final report, saves to Firestore

Connect: observe → detect → prioritize → fix → verify → report

State schema (TypedDict):
user_id, url, screenshot_base64, raw_violations, 
prioritized_violations, fixes_applied, score_before, 
score_after, scan_id, report

BACKEND — fix_generator.py

Function: generate_fix_action(violation: dict) -> dict

For each category generate the exact JavaScript DevTools command:
- COLOR_CONTRAST: change background/text color to pass 4.5:1 ratio
- FONT_SIZE: increase font-size to 16px minimum
- MISSING_ALT_TEXT: setAttribute('alt', 'descriptive text')
- KEYBOARD_FOCUS: add outline: '3px solid #005FCC'
- BUTTON_SIZE: set min width/height to 44px
- LINK_TEXT: suggest descriptive anchor text replacement
- FORM_LABELS: inject aria-label attribute

Return:
{
  'violation_id': str,
  'fix_applied': bool,
  'devtools_command': str,
  'explanation': str,
  'wcag_criterion_met': str,
  'compliance_improvement': int
}

FRONTEND — app.py (Streamlit)

4 pages via sidebar navigation:

Page 1 — Scan
- URL text input
- Screenshot file uploader (png/jpg/webp)
- "Scan for Violations" button
- POST to /scan, show spinner, display results

Page 2 — Results
- 4 metric cards: score before, score after, 
  violations found, fixes generated
- Violations list with severity color coding
- Each violation shows: category, WCAG criterion, 
  description, fix explanation
- DevTools commands in st.code() blocks
- Download full report as JSON button

Page 3 — Fix Executor
- Step-by-step instructions to apply fixes in Chrome DevTools
- All fix commands in copyable st.code() blocks
- Before/After comparison cards

Page 4 — History
- GET /history/{user_id}
- Show last 10 scans as expandable cards
